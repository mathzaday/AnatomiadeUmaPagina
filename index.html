Uma página que explica a si mesma

Bem-vindos! Esta página foi construída com um exemplo prático do uso de tags semânticas do HTML5. Cada seção que você vê abaixo não apenas utiliza as tags, mas também explica o por quê do seu uso. Inspecione o código para ver a mágica acontecer!

Estrutura Semântica

Pense nas tags de estrutura como as caixas transparentes de um mímico. Elas existem, definem espaços e organizam a cena, mas são invisíveis para o público. Elas não mudam o visual sozinhas, mas informam ao navegador o papel de cada bloco, melhorando a acessibilidade e o SEO.

Tags como code <header>, <main>, <section>, <article>, <aside>, <footer> formam o esqueleto da página.

<header> (Cabeçalho)

<article>

Este é o conteúdo principal e autônomo. Poderia ser lido separadamente e ainda faria sentido. Por exemplo, a matéria de um jornal.

<aside>

Isto é um conteúdo lateral. Relacionado ao <article>, mas não faz parte do fluxo principal. Pense em uma caixa de "leia também".

<footer> (Rodapé)

Agrupando Mídias e Legendas

A tag <figure> é usada para encapsular a imagem e sua legenda. Dentro dela, a tag <picture>, com suas tags <source>, otimiza a experiência visual, carregando a imagem ideal para cada dispositivo. A legenda é definida pela tag <figcaption>.

Para mídias como áudio e vídeo, o HTML5 oferece as tags <audio> e <video>, que criam players nativos no navegador. A tag <track> pode ser usada dentro delas para adicionar legendas, tornando o conteúdo acessível.
